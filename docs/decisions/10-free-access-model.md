# ADR 10: Free Access Model

## Status
Accepted

## Context
All capabilities should be available without paid tiers.

## Decision
No paywall features. All core and advanced functions are free.

## Technical Requirements
- Do not gate playback, themes, languages, or filters by subscription.
- Keep UX free from premium-lock messaging.

## Consequences
- Cost control depends on static hosting and browser-first architecture.
- Future monetization must be non-blocking if introduced (for example donations).
